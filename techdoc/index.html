Overview

WP-CLI is a command line tool that can be install on most *NIX based servers to install and manage WordPress sites.

WP-CLI is a community effort, which you can read about on their website.

Assumptions

This guide assumes you have the following basic background:

  • Comfortable with PHP/MySql
  • Comfortable with Command Line
  • You have installed WordPress manually via FTP or a File Manager.
  • Working knowledge of Shell/Terminal on UNIX machines or servers.
  • You have, or pay for, a webserver with SSH access.
    • Why use WP-CLI?

    You can quickly update all your themes and plugins and even script this in bash to cycle through all your WordPress installation directories. You can script new WordPress installs, add users and perform bulk edits to your WordPress database(s)... and Dry Run the operations to see results before committing them.

    Installing WP-CLI
    the "$" stands for your command prompt so only copy paste the command after the $ and hit "enter" for each line. There are 4 steps to install WP-CLI: $ curl -O https://raw.githubusercontent.com/wp-cli/builds/gh-pages/phar/wp-cli.phar $ php wp-cli.phar --info $ chmod +x wp-cli.phar $ sudo mv wp-cli.phar /usr/local/bin/wp First you use curl to download the wp-cli.phar file, then you verify it. The command chmod +x makes the file executable and sudo mv moves the file to a place more than likey listed in your PATH.
    Updating WP Core

    You can quickly update your WordPress install(s) by navigating to your WordPress install and running a simple command.

    $ cd ~/this/is/usually/public_html/wordPressSite/ $ wp core check-update $ wp core update

    First we navigated to the WordPress installation directory, then we checked for an update to the core and ran the update if there was one.

    Updating Plugins

    You can List, Enable, Disable and Update Plugins:

    $ wp plugin list $ wp disable plugin [[plugin name]] $ wp plugin update --all
    Updating Themes

    You can List, Enable, Disable and Update Plugins:

    $ wp theme list $ wp disable theme [[plugin name]] $ wp theme update --all
    Installing Core

    You can Install WP Core quickly and even hide the admin password from the bash history (see reference):

    $ wp core install --url=example.com --title=Example --admin_user=supervisor --admin_password=strongpassword --admin_email=info@example.com
